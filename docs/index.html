Do referee crews leave money on the total?

A leakage-safe backtest of public NBA officiating crews against prediction-market game totals.

01 · Thesis

Public info. Liquid markets. Classic underreaction bet.

Some crews call more fouls, slow the pace, push free-throw attempts. Assignments post before tip. If Kalshi or Polymarket underweight that, over/under prices should be wrong in a predictable direction.

Falsifiable claim Crew features improve out-of-sample P(over) vs a market + team baseline.

03 · Anti-leakage

The design fights cheating harder than it hunts an edge.

  • Rolling refs only — foul/FTA/pace from games strictly before tip
  • Shrinkage — thin histories pull toward a time-varying league prior
  • Market line as control — model hunts incremental signal, not the whole total
  • Walk-forward seasons — train N…N+k, test N+k+1. No random splits
  • Permutation test — shuffle crew labels; “edge” must disappear
  • Pre-tip prices — last print before tip-off, never settlement

05 · Outcome

No edge.

Crew features did not beat the market+team baseline out of sample. The permutation test says the tiny gap is noise.

Δ log-loss +0.00038 treatment worse
Permutation p 0.00 indistinguishable from null
Crews seen once 90% crew-level samples are thin

An honest null is the deliverable. Markets don’t appear to leave this public signal on the table under this design.
